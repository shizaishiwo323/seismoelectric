彬，我仔细复核后，结论是：

**现在这版代码已经比上一版严格很多，基本去掉了“Ricker + 电偶极子几何因子”的 reduced 近似，主线已经改成“Schakel 6×6 系数 + Liu 频率—波数积分”。但是还不能完全放心地说 100% 严格复现 Liu 2018，因为相位符号/时间约定这里仍需要统一。**

## 总体判断

**Schakel & Smeulders 2010 部分：核心公式基本是严格对应的。**

代码里的 `dynamic_coefficients()` 按 Schakel 附录 A 计算了动态渗透率 (k(\omega))、(\Lambda)、Debye 长度、(L(\omega))、(\sigma_f)、(C_{\rm em})、(C_{\rm os})、(\sigma(\omega)) 和 (\bar{\varepsilon})。这和 Schakel 论文中从 Pride 理论出发、用动态渗透率和电动耦合系数描述震电耦合的主线一致。  Schakel 论文正文也明确给出了 Biot 弹性系数、有效介电常数、有效密度、纵波/横波慢度以及 (R_E,T_{TM}) 的边界值问题。

**Liu et al. 2018 部分：框架上已经接近严格的频率—波数积分。**

现在代码的说明明确写了：波形只用 Liu 2018 风格的 frequency-wavenumber spectral integral，早期的 reduced shortcut 已经移除；峰值曲线也通过每个溶解时刻运行 spectral synthesis 计算；(k_b) 也改成显式的 `source_kb_m_inv` 参数。 这和 Liu 2018 里用有限宽度声源 (A(k,\omega)=\exp[-(k-k_0)^2/k_b^2]S(\omega))，再乘 (R_E(k,\omega)) 或 (T_E(k,\omega)) 做双重积分的思路是一致的。

## 已经做对的地方

第一，**reduced 模式确实被删除了**。现在没有 `synthesize_waveforms_reduced()`，也没有 `--waveform-mode reduced` 这类路径。顶部说明也明确说 earlier reduced Ricker + electric-dipole shortcut has been removed。

第二，**(R_E) 和 (T_{TM}) 不是经验因子，而是每个 ((k,\omega)) 点都调用 Schakel 6×6 边界值系统计算**。`se_coefficients()` 里先根据 (k_x) 或入射角计算水平波数，再计算 (k_3^{fl},k_3^E,k_3^{Pf},k_3^{Ps},k_3^{TM},k_3^{SV})，然后组装 6×6 矩阵求解 (R_E,R_M,T_{Pf},T_{Ps},T_{TM},T_{SV})。这对应 Schakel 论文式 45–46 里对反射/透射系数的定义和线性系统。 代码矩阵的 B2–B7 主体也确实按 Appendix B 在写。 

第三，**Liu 的有限宽度声源现在是显式 (k_b)**。代码中 `source_kb_m_inv = 80.0`，并在积分里使用 `Akw = source_pressure_amp * S_omega * exp(-((kx-k_center)/kb)**2)`，这比之前用 `1/beam_width` 间接代替 (k_b) 更接近 Liu 2018。 

第四，**峰值曲线也改成 spectral 计算了**。`compute_peak_amplitude_spectral()` 对每个有效溶解时刻调用 `synthesize_waveforms_spectral()`，不再用 (|R_E|)、(|T_{TM}|) 或偶极子几何因子做代理量。

第五，**T0 前响应被强制置零**。代码中 `U_real[:, t < T0] = 0.0`，这解决了你之前指出的“界面波不应该在声波到达界面前产生”的物理问题。 这个处理和 Liu 2018 中所有界面 EM 波到达时间基本等于 (T_0=z_s/V_f) 的解释是一致的。

## 还没有完全严格的地方

最关键的是**相位符号和时间约定**。

Schakel 2010 明确采用 ( \exp(i\omega t)) 时间约定。 Liu 2018 的式 1–2 按论文公式是以指数传播相位表示 ((k_z^iz_s \mp k_z^Ez + kx-\omega t)) 的频率—波数积分。 但当前代码文档和实现用的是：

[
\exp\left[i(k_z^iz_s-k_z^Ez+kx-\omega t)\right]
]

也就是代码里：

```python
phase_t_cache = exp(-1j * omega * t)
common_x = exp(1j * kx * x)
amp_z = exp(1j * (...))
```

这在**幅值包络**上可能不会马上暴露问题，但对**相位、极性、衰减方向、波形细节**是敏感的。尤其你现在要判断“界面上下极性反转”“有限偏移距波峰变化”“T0 后波形形态”，这个符号约定必须完全统一。否则会出现一种危险情况：图像趋势看着像，但公式约定并没有和论文完全一致。

第二个问题是：**代码做的是有限数值积分，不是无限域解析积分。** Liu 的积分理论上是 (\iint dk,d\omega)，而代码实际只积分 (0.25f_0\sim2.5f_0)，并且 (k) 被限制在传播声波分量附近，同时又限制在 (k_0\pm4k_b)。这作为数值实现是合理的，但论文里要写成“numerical frequency–wavenumber integration with finite spectral windows”，不能写成完全无截断积分。

第三个问题是：**因果 Ricker 源函数是你的合理改造，不是 Liu 原文逐字实现。** Liu 2018 只说用了 500 kHz Ricker wavelet；现在代码用有限时间窗、taper 和延迟的 causal Ricker spectrum，这是为了物理因果性做的改进。这个改动我支持，但方法部分要说明。

第四个问题是：**反应输运的 (H^+\rightarrow C,pH,\zeta,\sigma_f) 映射有模型假设。** 代码写得很诚实：如果没有实测电解质浓度，就用背景浓度加 (H^+) 近似；如果有 `FluidConductivity_S_m`，就覆盖 (\sigma_f)。 但这里会产生一个混合模型：(\sigma_f) 用实测值，而 (C_{\rm em},C_{\rm os}) 仍然由估算的离子浓度 (C) 参与计算。这个不是 Schakel 原文的“标准参数闭合”，而是反应输运耦合时的工程近似。

第五个问题是：**Schakel 验证还只是趋势验证，不是严格数值复现。** 代码已经加入 `schakel_standard_config()` 和 `run_schakel_validation()`，用 Table I 参数做角度扫描和频率扫描。 但代码自己也写了这是 “numerical self-check rather than a digitized figure match”。 所以目前还不能说已经严格复现 Schakel Fig. 2/3，只能说具备了验证入口。

## 我建议你马上改的唯一关键点

先统一相位约定。建议二选一：

**方案 A：严格跟 Liu/Schakel 的 (e^{i\omega t}) 约定走。**

那么波形积分可以写成：

```python
phase_t = np.exp(1j * omega * t)
common_x = np.exp(-1j * kx * x)

fluid:
amp = RE * np.exp(-1j * (kzi * z_s - kEr * z))

porous:
amp = TTM * np.exp(-1j * (kzi * z_s + kEt * z))
```

同时，`complex_sqrt_branch()` 要配合这个传播约定，保证传播方向上是衰减而不是增长。

**方案 B：保持现在代码的 (e^{-i\omega t}) 数值约定。**

那就必须在方法说明里明确声明：本代码采用与 Schakel 论文相反的 Fourier 时间约定，并且所有频域系数、动态渗透率、电导率和传播波数都按该约定做了共轭一致化。现在代码还没有完全做到这点，所以我不建议选 B。

## 最终可用性判断

你现在这版可以比较有把握地说：

> The seismoelectric reflection coefficient (R_E) and TM-mode transmission coefficient (T_{TM}) are computed using the Schakel and Smeulders fluid–porous medium boundary-value formulation. The interface EM waveforms are then synthesized through a Liu et al.-style finite-offset frequency–wavenumber integral with a finite-width acoustic source.

但暂时不要写：

> The code strictly reproduces Liu et al. (2018).

更严谨的说法是：

> The code implements the core theoretical framework of Liu et al. (2018) and Schakel and Smeulders (2010), with finite numerical spectral windows and a causal Ricker source modification. A remaining convention check is required for the global Fourier phase sign.

**一句话评价：这版已经从“近似演示代码”升级到了“接近论文理论主线的 spectral 正演代码”；但在你正式用于论文结果前，必须先把相位符号约定统一掉，再做 Schakel 标准图的定量复现和 (n_\omega,n_k) 收敛测试。**
